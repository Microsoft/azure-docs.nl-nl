# <a name="quotas-in-azure-static-web-apps-preview"></a>Quota in Azure Static Web Apps Preview

De volgende quota bestaan voor Azure Static Web Apps Preview.

> [!IMPORTANT]
> Azure Static Web Apps is in de openbare preview-versie en is niet bedoeld voor gebruik in productie.

| Functie                     | Gratis abonnement        |
|-----------------------------|------------------|
| Inbegrepen bandbreedte          | 100 GB per maand |
| Overschrijding van bandbreedte           | Niet beschikbaar      |
| Apps per Azure-abonnement | 10               |
| App-grootte                    | 250 MB           |
| Preproductieomgevingen | 3                |
| Aangepaste domeinen              | 1                |
| Autorisatie (met aangepaste rollen en routerings regels) | Maximum van 25 eind gebruikers die deel kunnen uitmaken van aangepaste rollen |
| Azure Functions             | Beschikbaar        |
| SLA                         | Geen             |

## <a name="github-storage"></a>GitHub-opslag

GitHub-opslag heeft een eigen set quota en wordt gebruikt voor GitHub-acties en -pakketten. Wanneer u een Azure Static Web Apps-site maakt, worden in GitHub de artefacten voor de site opgeslagen, zelfs na de implementatie.

Zie de volgende resources voor meer details:

- [Opslagruimte voor acties beheren](https://github.community/t5/GitHub-Actions/Managing-Actions-storage-space/td-p/38944)
- [Over facturering voor GitHub-acties](https://help.github.com/github/setting-up-and-managing-billing-and-payments-on-github/about-billing-for-github-actions#about-billing-for-github-actions)
- [Uw bestedingslimiet voor GitHub-acties beheren](https://help.github.com/github/setting-up-and-managing-billing-and-payments-on-github/managing-your-spending-limit-for-github-actions)
